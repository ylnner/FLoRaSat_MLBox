Satellite Network Layer

Overview

The satellite network layer handles packet routing and forwarding for LEO satellite nodes. It uses a modular architecture separating statistics collection, packet dispatching, and routing algorithms.

Architecture

Sat_Net (Compound Module)
├── @class: Sat_Net_SatelliteRouting (Statistics via signals)
├── packetHandler: Sat_Net_PacketHandler (Central dispatcher)
└── routing: <routingType> (e.g., StoreAndForward)

Key Components:

  • Sat_Net.ned: Compound module with pass-through gate connections to packetHandler
  • Sat_Net_SatelliteRouting: Statistics collector attached via @class annotation
  • Sat_Net_PacketHandler: Receives all packets, delegates to routing, executes decisions with per-gate queuing
  • Sat_Net_PacketSender: Interface for routing modules to send/drop packets
  • StoreAndForward: Example routing implementation (store when isolated, forward when GS visible)
  • Sat_Net_SdrModel: FIFO buffer for store-and-forward packet storage

Packet Flow

  1. Packet arrives at input gate (ISL/GSL/DSL) → routed to packetHandler via NED connections
  2. PacketHandler::handleMessage() → emit signals, forward to routing module
  3. routing->handlePacket(pkt) → analyze and decide direction
  4. Routing calls back sender->sendMessage(pkt, ISLDirection, silent, gsIndex)
  5. PacketHandler maps direction to gate, queues if busy, sends when free
  6. SatelliteRouting captures signals and updates statistics

Adding Custom Routing

Create MyRouting.h/cc/ned in this folder:

// MyRouting.h
class MyRouting : public Base_Routing {
public:
virtual void handlePacket(inet::Packet* pkt) override;
virtual void handleTopologyChange(bool changes) override;
};

Configure in .ini: **.satellite[*].netLayer.routingType = "MyRouting"
