Satellite Node

Overview

The Satellite node represents LEO satellites providing connectivity backbone for IoT networks. It handles packet forwarding, inter-satellite links (ISL), ground-satellite links (GSL), and device-satellite links (DSL).

Architecture

Sat (Compound Module)
├── DSL/: Device-Satellite Link (LoRa downlink/uplink)
├── GSL/: Ground-Satellite Link (to ground stations)
├── ISL/: Inter-Satellite Link (satellite mesh network)
├── _30_Network/: Routing and packet forwarding
├── _40_Application/: Satellite services and control
├── _50_Energy/: Solar power and battery management
└── _60_Mobility/: Orbital mechanics (TLE-based)

Key Components:

  • Sat.ned: Compound module with multiple radio interfaces (DSL, GSL, ISL)
  • Sat.h/cc: Container module for satellite node
  • Link Layers: DSL, GSL, ISL each have dedicated PHY/MAC

Packet Flow

  1. Uplink: Terminal → DSL → Network → ISL/GSL
  2. Transit: ISL → Network (routing) → ISL
  3. Downlink: ISL/Network → GSL → Ground Station

Configuration

Satellite parameters in .ini:

*.satellite[*].dsl.radio.transmitter.power = 27dBm
*.satellite[*].isl.numRadios = 4 # Four ISL radios
*.satellite[*].netLayer.routingType = "StoreAndForward"

Implementation Details

  • Multiple possible radio interfaces (DSL, GSL, ISL)
  • Dynamic routing based on topology control
  • Store-and-forward when isolated
  • TLE-based orbital propagation
