Utilities

Overview

The Utilities folder provides shared infrastructure, helper functions, and core abstractions used throughout the simulator.

Architecture

Utilities/
├── Constants.h: Physical constants and default values
├── FloraSignals: OMNeT++ signal definitions
├── ISLDirection: Inter-satellite link direction enums
├── ISLState: Link state tracking
├── LayerForwarder: Cross-layer message passing
├── PositionAwareBase: Geolocation utilities
├── Timer.h: Event scheduling helpers
├── Utils: Math and conversion functions
├── WalkerType: Constellation pattern definitions
├── Routing_Core/: Dijkstra and graph algorithms
├── libnorad/: SGP4 orbital propagation library
└── utils/: Additional helper modules

Key Components:

  • FloraSignals: Centralized signal registry for cross-module communication
  • ISLDirection: Enumerations for link directions (INTRA_PLANE, INTER_PLANE, UP, DOWN)
  • LayerForwarder: NED module for transparent message forwarding between layers
  • PositionAwareBase: Provides lat/lon/alt accessors and distance calculations
  • Routing_Core/: Graph-based routing algorithms (Dijkstra, Floyd-Warshall)
  • libnorad/: Third-party SGP4/SDP4 implementation for TLE orbit propagation

Usage

Utilities are header-only or minimal implementations included throughout:

#include "Global/Utilities/Utils.h"
double distance = calculateDistance(coord1, coord2);
Utils.h

Implementation Details

  • Constants defined in SI units (meters, seconds, Hz)
  • Signals registered at simulation start via registerSignal()
  • Routing algorithms operate on adjacency matrices
  • Position-aware modules inherit from PositionAwareBase
