Terminal Application Layer

Overview

The terminal application layer generates and consumes traffic for IoT scenarios, modeling sensor data, telemetry, and control messages.

Architecture

_40_Application/
├── Ter_App: Application logic and traffic generation
└── Ter_App.ned: Application module definition

Key Components:

  • Ter_App: Traffic generator and sink
    • Periodic data transmission
    • Event-driven messaging
    • Downlink message handling

Traffic Patterns

  • Periodic: Regular sensor updates (e.g., every 60s)
  • Event-driven: Triggered by external events
  • Burst: Multiple packets in rapid succession

Configuration

*.terminal[*].app.typename = "Ter_App"
*.terminal[*].app.sendInterval = 60s
*.terminal[*].app.packetLength = 50B
*.terminal[*].app.destAddress = "server"

Implementation Details

  • Generates LoRaAppPacket with payload and headers
  • Records end-to-end delay and delivery statistics
