LoRa Transceiver

Overview

The Transceiver folder implements LoRa physical layer transmission and reception logic, including collision detection, interference handling, and modulation-aware signal processing.

Architecture

Transceiver/
├── LoRaTransmitter: Encodes packets into LoRa transmissions
├── LoRaReceiver: Decodes receptions, handles collisions
├── LoRaTransmission: Represents transmitted LoRa signal (SF, BW, CR, power)
└── LoRaReception: Represents received signal with arrival time

Key Components:

  • LoRaTransmitter: Extends FlatTransmitterBase, creates LoRaTransmission objects with modulation parameters
  • LoRaReceiver: Extends FlatReceiverBase, centralizes collision handling via nonOrthDelta matrix
  • LoRaTransmission/Reception: Carry LoRa-specific parameters (spreading factor, coding rate, bandwidth)

Collision Handling

The receiver evaluates interference based on:

  • Orthogonality: Different spreading factors can coexist (configurable via nonOrthDelta)
  • SNIR Thresholds: Configurable per spreading factor (snirThresholdSF7-SF12)
  • Capture Effect: Stronger signals can suppress weaker ones (ignoreInterference)

Configuration

Set transceiver parameters in .ini:

**.radio.transmitter.typename = "LoRaTransmitter"
**.radio.receiver.typename = "LoRaReceiver"
**.radio.receiver.snirThresholdSF7 = -20dB

Implementation Details

  • Uses nonOrthDelta matrix to model inter-SF interference
  • Integrates with LoRaAnalogModel for power and noise computation
  • Supports capture effect when SNIR exceeds thresholds
