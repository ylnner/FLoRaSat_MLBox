Terminal Node

Overview

The Terminal node represents IoT devices and end-user equipment communicating via LoRa with LEO satellites. It implements the full protocol stack from application to physical layer.

Architecture

Ter (Compound Module)
├── _10_Physical/: LoRa radio and PHY layer
├── _20_Data_Link/: kMac MAC protocol
├── _30_Network/: Routing and forwarding
├── _40_Application/: Traffic generation and consumption
├── _50_Energy/: Battery and power management
└── _60_Mobility/: Terminal movement patterns

Key Components:

  • Ter.ned: Compound module definition with gate connections
  • Ter.h/cc: Container module for terminal node
  • Layer Modules: Each layer folder contains protocol implementations

Packet Flow

  1. Application generates data → Network layer
  2. Network attaches routing headers → MAC
  3. MAC encapsulates and schedules transmission → PHY
  4. PHY modulates and transmits LoRa signal

Configuration

Terminal parameters in .ini:

*.terminal[*].app.destAddress = "server"
*.terminal[*].mac.LoRaSF = 7
*.terminal[*].radio.transmitter.power = 14dBm

Implementation Details

  • Subclasses extend base protocols from Global/Base/
  • Uses LoRa modulation for satellite uplinks
  • Supports mobile and stationary deployments
  • Integrates with energy models for battery constraints
