Terminal Data Link Layer

Overview

The terminal data link layer implements the kMac protocol, a lightweight MAC for LoRa terminals with scheduled transmission windows.

Architecture

_20_Data_Link/
├── Ter_Dlk_kMac: kMac protocol implementation
└── Ter_Dlk.ned: Compound module for data link layer

Key Components:

  • Ter_Dlk_kMac: Extends Base_MacProtocol
    • KIWAN_IDLE: Default state, waiting for transmission opportunity
    • KIWAN_TRANSMIT: Sending frame to satellite

MAC Protocol

kMac operates with simple ALOHA-style access:

  1. Application data arrives at MAC
  2. MAC schedules transmission
  3. Frame encapsulated with kMacFrame header
  4. Transmission sent to PHY layer
  5. No acknowledgments (unconfirmed uplink)

Configuration

*.terminal[*].mac.typename = "Ter_Dlk_kMac"
*.terminal[*].mac.LoRaSF = 7
*.terminal[*].mac.LoRaBW = 125000
*.terminal[*].mac.headerLength = 20B

Implementation Details

  • Queues packets when radio is busy
  • Adds MAC headers with source/destination addresses
  • Coordinates with PHY for transmission timing
