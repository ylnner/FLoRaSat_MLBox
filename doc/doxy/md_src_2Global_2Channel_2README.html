Channel Models

Overview

The Channel folder provides LoRa-specific propagation and path-loss models for satellite-to-ground and inter-satellite communications. It implements custom analog models extending INET's physical layer.

Architecture

Channel/
├── LoRaAnalogModel: Custom SNIR computation and noise handling
├── LoRaBandListening: LoRa-specific listening state
├── LoRaModulation: Spreading factor and coding rate definitions
├── Path Loss Models:
│ ├── LoRaHataOkumura: Urban/suburban propagation
│ ├── LoRaLogNormalShadowing: Statistical fading
│ └── LoRaPathLossOulu: High-latitude model
└── Propagation/: Delay and attenuation calculators

Key Components:

  • LoRaAnalogModel: Extends ScalarAnalogModelBase to compute reception power, background noise, and SNIR for LoRa transmissions
  • LoRaModulation: Defines modulation parameters (SF, BW, CR) and computes symbol/bit durations
  • Path Loss Models: Implement distance-dependent attenuation for various environments

Usage

Path-loss models are selected in .ini files:

*.radioMedium.pathLossType = "LoRaLogNormalShadowing"
*.radioMedium.analogModelType = "LoRaAnalogModel"

Implementation Details

  • SNIR computation accounts for LoRa's orthogonal spreading factors
  • Background noise derived from thermal noise and bandwidth
  • Models support both terrestrial and LEO-to-ground scenarios
