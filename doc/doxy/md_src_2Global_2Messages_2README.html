Message Definitions

Overview

The Messages folder contains OMNeT++ message (.msg) files defining packet structures and control information for all protocol layers.

Architecture

Messages/
├── _10_Physical/: PHY layer control info (radio params, channel state)
├── _20_Data_Link/: MAC frames, tags, control messages
├── _30_Network/: Routing headers, topology tags
└── _40_Application/: Application-level packets

Key Components:

  • **_10_Physical/**: LoRaRadioControlInfo_m - transmission parameters (SF, BW, CR, power)
  • **_20_Data_Link/**: Base_MacFrame_m, kMacFrame_m, LoRaTagInfo_m - MAC headers and tags
  • **_30_Network/**: Net_CstRoutingTag_m - routing metadata (next hop, path)
  • **_40_Application/**: LoRaAppPacket_m - application payload and headers

Message Flow

  1. Application creates LoRaAppPacket
  2. Network attaches Net_CstRoutingTag for routing
  3. MAC encapsulates in kMacFrame or similar
  4. PHY adds LoRaRadioControlInfo for transmission

Implementation Details

  • .msg files compiled by opp_msgc to generate _m.h and _m.cc
  • Messages inherit from inet::Packet or cMessage
  • Tags attach metadata without modifying packet structure
  • Control info passed via setControlInfo() for cross-layer communication
