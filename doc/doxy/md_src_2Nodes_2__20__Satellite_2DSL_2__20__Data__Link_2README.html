DSL Data Link Layer

Overview

The DSL data link layer implements satellite-side MAC protocols for coordinating communications with IoT terminals.

Architecture

_20_Data_Link/
└── Sat_Dsl_Dlk.ned: DSL MAC configuration

Key Components:

  • DSL MAC: Coordinates terminal access
    • Beacon transmission (downlink)
    • Random access (uplink)
    • Optional slotted ALOHA
    • Frame acknowledgment (optional)

Protocol Operation

  1. Beacon Phase: Satellite broadcasts beacon with timing info
  2. Uplink Phase: Terminals transmit data (ALOHA or slotted)
  3. Reception: Satellite receives and forwards to network layer
  4. Downlink Phase: Optional unicast/multicast to terminals

Configuration

*.satellite[*].dsl.mac.beaconInterval = 10s
*.satellite[*].dsl.mac.beaconSF = 7
*.satellite[*].dsl.mac.enableACK = false
*.satellite[*].dsl.mac.uplinkSlots = 100

Implementation Details

  • Beacons announce satellite presence and timing
  • Uplink collisions resolved via capture effect
  • No ACKs by default (unconfirmed uplink for energy efficiency)
  • Can implement Class B (scheduled downlink) for confirmed delivery
